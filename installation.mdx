# Installation

OpenCompany runs locally with the `company` CLI. For production, deploy to a cloud VM with `company deploy`.

## Prerequisites

- Node.js 22+
- Python 3.12+
- [uv](https://docs.astral.sh/uv/) (Python package manager)
- Git

## Quick Install

<Tabs>
  <Tab title="Install the CLI">
    Install OpenCompany globally and start it:

    ```bash
    npm install -g @zeenie-ai/opencompany
    company start
    ```

    The app will be available at:
    - Frontend: `http://localhost:3000`
    - Backend: `http://localhost:3010`
    - Node.js executor: `http://localhost:3020`
    - WhatsApp: `http://localhost:9400`
    - Temporal Web UI: `http://localhost:8080`

    <Tip>
      After the global install the binary is `company`. The old `machina` command still exists only as a deprecated alias.
    </Tip>
  </Tab>
  <Tab title="From source">
    ```bash
    # Clone the repository
    git clone https://github.com/zeenie-ai/opencompany.git
    cd opencompany

    # Build (installs client + server dependencies via uv)
    npm run build

    # Start all services (production mode)
    npm run start

    # Or start with hot-reload for development
    npm run dev
    ```

    Under the hood the npm scripts call the `company` CLI (`python -m cli start` / `dev` / `build`). The backend is managed by [uv](https://docs.astral.sh/uv/) (`uv sync` against `server/uv.lock`), not pip.
  </Tab>
</Tabs>

## CLI Commands

The `company` CLI drives every lifecycle action (`machina` still exists only as a deprecated alias):

| Command | Description |
|---------|-------------|
| `company start` | Start all services (production mode, clean output) |
| `company dev` | Start with hot-reload for development (`--force` re-bundles Vite deps, `--daemon` binds the backend to 0.0.0.0) |
| `company build` | Install dependencies and build for production |
| `company serve` | Serve API + WebSocket + SPA on a single public port |
| `company stop` | Stop all running services |
| `company clean` | Clean build artifacts |
| `company deploy` | Provision a cloud VM running OpenCompany (`up` / `status` / `destroy`) |
| `company daemon` | Run the backend as a detached process (`start` / `stop` / `status` / `restart`) |
| `company version sync` | Sync package versions from the latest git tag |
| `company docs nodes` | Rebuild (or `--check`) the per-node documentation index |

Useful flags: `--verbose` / `-v` (full service logs), `--skip-whatsapp` (skip the WhatsApp service).

## Environment Configuration

Copy the environment template:

```bash
cp .env.template .env
```

### Key Settings

<ParamField path="VITE_CLIENT_PORT" type="number" default="3000">
  Frontend development server port
</ParamField>

<ParamField path="PYTHON_BACKEND_PORT" type="number" default="3010">
  Backend API server port
</ParamField>

<ParamField path="AUTH_MODE" type="string" default="single">
  Authentication mode: `single` (first user is owner) or `multi` (open registration)
</ParamField>

<ParamField path="REDIS_ENABLED" type="boolean" default="false">
  Enable Redis cache. Set to `true` for production.
</ParamField>

<ParamField path="VITE_ANDROID_RELAY_URL" type="string">
  WebSocket URL for remote Android device connections
</ParamField>

### Full Environment Example

```bash
# Ports
VITE_CLIENT_PORT=3000
PYTHON_BACKEND_PORT=3010

# Authentication
AUTH_MODE=single
JWT_SECRET_KEY=your-secret-key-min-32-chars

# Cache (production)
REDIS_ENABLED=false
REDIS_URL=redis://localhost:6379

# Optional API Keys
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
GOOGLE_AI_API_KEY=
GOOGLE_MAPS_API_KEY=
```

## Verify Installation

1. Open `http://localhost:3000` in your browser
2. You should see the OpenCompany workflow editor
3. Check the backend health: `curl http://localhost:3010/health`

<Check>
If you see the workflow canvas with the component palette, installation is complete!
</Check>

## Production Deployment

The recommended way to run OpenCompany in production is `company deploy`, which provisions a login-gated cloud VM (currently Google Cloud) using your cloud CLI plus Terraform, then runs OpenCompany on a single port via `company serve` under systemd:

```bash
company deploy up --provider gcp --owner-email you@example.com   # provision + install + print URL/creds
company deploy status                                            # show URL + health
company deploy destroy                                           # tear down
```

## Troubleshooting

<Accordion title="Port already in use">
  Change the port in `.env`:
  ```bash
  VITE_CLIENT_PORT=3001
  PYTHON_BACKEND_PORT=3011
  ```
  OpenCompany also uses ports 3020 (Node.js executor), 9400 (WhatsApp), 7233 (Temporal gRPC), 8080 (Temporal Web UI), and 6379 (Redis, when enabled).
</Accordion>

<Accordion title="Python dependencies fail">
  The backend is managed by [uv](https://docs.astral.sh/uv/). Sync the environment from the lock file:
  ```bash
  cd server
  uv sync
  ```
  This creates `server/.venv` against `server/uv.lock`. Make sure `uv` is installed and on your PATH first.
</Accordion>

<Accordion title="Services won't start">
  Stop everything cleanly, then start again with full logs:
  ```bash
  company stop
  company start -v
  ```
  If a build looks stale, run `company clean` before `company build`.
</Accordion>
